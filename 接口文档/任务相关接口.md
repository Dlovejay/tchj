### 添加任务状态 - task/add

请求参数：

|名称|类型|必填|说明|可选值|
|----|:--:|:--:|----|----|
|title|string|Y|标题||
|content|string|Y|内容||
|start_at|int|Y|开始时间||
|end_at|int|N|结束时间|为0表示无限|
|annex|json|N|附件|不传为空|
|departments|string|Y|部门id|多个以逗号分隔<br>0表示全部部门<br>数据库里面也这么写|
|initiate_pid|int|N|发起部门id||

annex类型:

    [
        {
            "type":1, //1:ftp 2:其它
            "url": "xxxx",
            "name": "xeks" //name 由前端解析传入
        }
    ]


返回：

    {
        "code": 0,
        "message":"",
        "data": {
            "id":123,
        }
    }
    
### 获取任务列表 task/lists

请求参数：

|名称|类型|必填|说明|可选值|
|----|:--:|:--:|----|----|
|keywords|string|N|关键字||
|status|string|N|状态||
|is_timeout|string|N|状态|TRUE, FALSE, 不传为全部|
|page|int|N|页码|默认第一页|
|pagesize|int|N|每页显示个数|默认15个|

                    "end_at":1234,
                    "update_at":12345,
                    "create_at":1234,
                    "departments":[
                        1,2,3
                    ],
                    "create_user_id": 12,
                    "last_do_user_id":13,
                    "count": 2, //回复数
                    "status":"1",
                    "annex":[],
                    "is_timeout":true|false,
                    "initiate_pid":1
                },
                {},
                ...
            ]
        }
    }
    
    
### 获取任务明细 task/detail

请求参数：

|名称|类型|必填|说明|可选值|
|----|:--:|:--:|----|----|
|id|int|Y|任务id||

返回：

    {
        "code": 0,
        "data":{
            "replys"[
                {
                    "reply_id": 1,
                    "create_user_id": 12,
                    "cause":xxxx,// 不过期无此选项
                    "content":"xxx",
                    "update_at":123445,
                    "status": "1",
                },
                {},
                ...
            ],// 回复列表
            "do":[
               "DELETE","EDIT","RECEIVE","REPLY","FINISHED","BACK","REPEAL"
            ],// 可用操作
            "is_timeout":true|false,//是否过期
            "can_edit": true|false //是否能修改
        }
    }

### 修改任务 task/edit

请求参数：

当 status=0|1

|名称|类型|必填|说明|可选值|
|----|:--:|:--:|----|----|
|id|int|Y|任务id||
|title|string|Y|标题||
|content|string|Y|内容||
|start_at|int|Y|开始时间||
|end_at|int|Y|结束时间||
|annex|json|Y|附件||
|departments|string|Y|部门id|多个以逗号分隔|
|initiate_pid|int|N|发起部门id||

返回：

    {
        "code":0
    }

### 任务流转 task/next

请求参数：

|名称|类型|必填|说明|可选值|
|----|:--:|:--:|----|----|
|id|int|Y|任务id||
|content|string|Y|内容||
|do|string|Y|操作|RECEIVE|

当 is_timeout = true 时，需要传原因

|名称|类型|必填|说明|可选值|
|----|:--:|:--:|----|----|
|cause|string|Y|过期原因||

返回值：

    {
        "code": 0,
        "message": "",
        "data": {
            "is_timeout": true,
            "do": [
                "REPLY"
            ],
            "canEdit": false,
            'status':2,
        }
    }


### 统计   task/statistics
请求参数：

    无输入参数

返回值：

    {
        "code": 0,
        "message": "",
        "data": [
            {
                "pid": 0,
                "name": "总计",
                "count": {
                    "total": 13, //总数
                    "doing": 10, //进行中
                    "repeal": 1, //撤销
                    "timeout": 9, //过期
                    "first_finish_percent": 50, //首次完成率
                    "reply_percent": 10 // 提交率
                }
            }
        ]
    }
    
ps:
    当 用户为管理员和领导时，数组中有多个元素，其它角色数组中只有一个元素



## 常量表：
### 状态
|code|说明|
|----|----|
|0|未开始|
|1|已开始|
|2|已接受|
|3|已回复|
|4|完成|
|5|打回|
|6|已删除|
|7|作废|


### 操作
|code|说明|
|----|----|
|RECEIVE|接受|
|REPLY|回复|
|FINISHED|完成|
|BACK|打回|
|DELETED|删除|
|EDIT|修改|
|REPEAL|作废（撤销）|
